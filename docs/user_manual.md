## 安装与首次设置

### 系统要求

- Windows 10/11、macOS 12+ 或 Ubuntu 20.04+
- 局域网连接（用于 P2P 组发现）

### 首次启动

1. 运行 `synapse-vault` 可执行文件。
2. 首次使用会进入**首次设置**界面：
   - 选择密钥文件（`.key`）的保存路径。
   - 设置主密码（至少 8 位）。
   - 确认主密码。
3. 点击"创建密钥并进入"。
   - 应用会自动校准 Argon2id 加密参数（约需几秒）。
   - 密钥文件生成后自动解锁进入主界面。

### 日常解锁

1. 运行应用后，输入密钥文件路径（默认会自动填充）。
2. 输入主密码。
3. 点击"解锁"。

## 创建组 / 发现并加入组

### 创建组（Admin）

1. 点击左侧导航栏的"📁 组管理"。
2. 点击"创建组"，输入组名和描述。
3. 组创建后，其他局域网设备可通过 mDNS 自动发现该组。

### 发现并加入组

1. 在"组管理"面板，点击"发现组"。
2. 列表会显示局域网内可加入的组。
3. 选择目标组，点击"申请加入"。
4. 等待 Admin 审批通过后，即可成为该组成员。

## 管理密码

### 创建密码

1. 进入"🔑 密码库"面板。
2. 点击"新建密码"，填写：
   - 标题、用户名、密码
   - 环境（如 dev/prod）
   - 标签（可选）
   - 描述（可选）
3. 点击保存。密码会自动同步到组内其他成员。

### 查看密码

- 在密码列表中点击目标条目，输入主密码后即可查看明文密码。
- AuditUser 查看密码前需要 Admin 审批（见[使用审批流程](#使用审批流程)）。

### 复制密码

- 点击密码条目旁的"复制"按钮，密码会被安全复制到剪贴板，30 秒后自动清除。

### 搜索密码

- 在密码库顶部的搜索框输入关键词，支持按标题、用户名、标签搜索。

### 删除密码

- 点击密码条目旁的"删除"按钮，确认后该密码会被标记为删除并同步到全组。

## 角色管理

SynapseVault 支持三种角色：

| 角色 | 权限 |
|------|------|
| **Admin** | 创建组、审批加入请求、管理成员角色、审批使用请求 |
| **FreeUser** | 查看/复制/创建/更新密码 |
| **AuditUser** | 查看审计日志、申请查看/复制密码（需 Admin 审批） |

### 变更角色

1. Admin 进入"🛡️ 权限"面板。
2. 在成员列表中选择目标成员。
3. 点击"变更角色"，选择新角色并确认。

## 审计日志查看与导出

### 查看审计日志

1. 进入"📋 审计"面板。
2. 使用"操作类型"下拉框筛选特定操作（如查看密码、复制密码、成员审批等）。
3. 点击"🔄 刷新"更新日志列表。

### 导出审计日志

1. 在审计面板点击"📤 导出 JSON"或"📤 导出 CSV"。
2. 选择保存路径，确认后即可导出。

---

## 使用审批流程

AuditUser 查看或复制密码前，需要 Admin 的临时使用审批。

### AuditUser 申请

1. 尝试查看或复制密码时，系统提示"权限不足"。
2. 点击"📋 申请查看/复制权限"。
3. 填写申请理由，提交后等待 Admin 审批。

### Admin 审批

1. Admin 在"🛡️ 权限"面板查看待审批的使用请求。
2. 点击"通过"或"拒绝"。
3. 通过的审批在有效期内（默认 5 分钟，可在设置中调整）允许 AuditUser 执行对应操作。

## 导入/导出密码

### 导出密码

1. 在密码库中，选择需要导出的条目（可多选）。
2. 点击"导出"，选择格式（JSON 或 CSV）。
3. 选择保存路径完成导出。

### 导入密码

1. 在密码库中点击"导入"。
2. 选择符合格式的 JSON 或 CSV 文件。
3. 确认导入后，密码会自动同步到组内。

## 设置

进入"⚙️ 设置"面板（点击顶部栏的齿轮图标）：

| 设置项 | 说明 |
|--------|------|
| 使用审批有效期 | AuditUser 使用审批的 TTL（分钟） |
| 自动锁定超时 | 闲置后自动锁定应用的时间（0=禁用） |
| 主题 | 深色 / 浅色模式切换 |
| Argon2id 参数 | 当前密钥文件的加密参数（只读） |
| 数据库路径 | 本地 SQLite 数据库位置（只读） |
| 密钥文件路径 | 当前使用的 `.key` 文件路径（可修改） |

## 安全最佳实践

1. **主密码强度**：使用至少 12 位，包含大小写字母、数字和符号的强密码。
2. **密钥文件备份**：将 `.key` 文件备份到安全位置（如加密 U 盘），不要与主密码放在同一处。
3. **自动锁定**：建议设置 10-30 分钟的自动锁定超时。
4. **局域网安全**：确保局域网可信，避免在公共 Wi-Fi 下使用。
5. **定期审计**：Admin 应定期查看审计日志，发现异常操作及时处理。

## 故障排除

### 忘记密码

1. 在解锁界面点击"忘记密码？"。
2. 确认重置后，本地密钥文件将被删除。
3. **注意**：重置后需要重新创建身份并申请加入组，本地数据无法恢复。

### 设备指纹不匹配

- 密钥文件与当前设备绑定。如果在另一台设备上使用该密钥文件，会提示"设备指纹不匹配"。
- 解决方案：在该设备上重新创建密钥文件，然后申请加入原组。

### 无法发现组

1. 确认所有设备在同一局域网内。
2. 检查防火墙是否阻止了 mDNS（端口 5353）或 libp2p 端口。
3. 尝试手动输入组管理员提供的连接地址。

### 同步失败

1. 检查网络连接是否正常。
2. 确认所有成员使用的是兼容的应用版本。
3. 查看审计日志中的错误信息，联系 Admin 排查。
